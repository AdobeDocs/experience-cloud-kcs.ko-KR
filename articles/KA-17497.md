# AEM Dispatcher가 캐시하는 요청을 알아봅니다

## 설명

AEM Dispatcher는 어떤 HTTP 요청 및 응답을 캐시합니까? (AEM Dispatcher 4.3.3)<br><br><br>

## 해상도


이 설명서는 AEM Dispatcher 모듈을 통해 전달되는 요청 및 응답의 캐시 능력에 영향을 주는 가장 중요한 시나리오를 다룹니다. 모든 캐시 규칙은 적용되지 않습니다.
<br><br><br><br> <br><br>HTTP 요청 및 캐싱 규칙:<br><br><br><br> <br><br>
HTTP 요청을 디스패처에서 캐시하려면 다음 규칙을 준수해야 합니다.
<br><br><br><br> <br><br>1. URL은 절대입니다(로 시작하지 않음). 또는 ..)에는 파일 확장자가 있고 URL의 파일 이름은 파일 시스템에서 파일로 작동합니다.<br><br><br><br> <br><br>
<b>관련 로그 메시지:</b>
<br><br><br><br><br> <br><br> <br><br><br><br>

| `URI not canonical: %s.``Unable to map URI to ``file``: %s.``cache ``file` `path too long``temporary ``file` `path too long``request URL has no extension` |
| --- |

<br><br><br><br><br> <br><br>
<b>예:</b>
<br><br><br><br> <br><br>

| /content/test | 캐시되지 않음 |
| --- | --- |
| /content/test.html | 캐시됨 |

<br><br><br><br> <br><br>2. 첫 번째 파일 확장자 뒤에 슬래시가 없습니다.<br><br><br><br> <br><br>
<b>관련 로그 메시지:</b>
<br><br><br><br><br> <br><br> <br><br><br><br>

| `request URL has trailing slash` |
| --- |

<br><br><br><br><br> <br><br>
<b>예:</b>
<br><br><br><br> <br><br>

| /content/test.html/ | 캐시되지 않음 |
| --- | --- |
| /content/test.html/foo.jpg | 캐시되지 않음 |
| /content/test.html | 캐시됨 |

<br><br><br><br> <br><br>3. HTTP GET 또는 HEAD 메서드를 사용합니다.<br><br><br><br> <br><br>
<b>관련 로그 메시지:</b>
<br><br><br><br><br> <br><br> <br><br><br><br>

| `request method is neither GET nor HEAD` |
| --- |

<br><br><br><br><br> <br><br>
<b>예:</b>
<br><br><br><br> <br><br>

| HEAD /content/test.html | 캐시됨 |
| --- | --- |
| GET /content/dam/test.jpg | 캐시되지 않음 |
| POST /content/test.html | 캐시되지 않음 |

<br><br><br><br> <br><br>4. DISPATCHER .ANY 팜 구성에 정의된 /CACHE = /RULES에 의해 요청이 거부됩니다.<br><br><br><br> <br><br>
<b>관련 로그 메시지:</b>
<br><br><br><br><br> <br><br> <br><br><br><br>

| `URI not ``in` `cache rules: %s``request URL not ``in` `cache rules` |
| --- |

<br><br><br><br><br> <br><br>
<b>예:</b>
<br><br><br><br> <br><br>

| 팜에 /cache = /rules가 있으며, 요청이 수신됩니다. <b>/content/test.html</b><br><br>/cache<br><br>{<br><br>...<br><br>/rules<br><br>{<br><br>/0001 {<br><br>&quot;deny&quot; 유형<br><br>glob &quot;\*&quot;<br><br>}<br><br>} | 캐시되지 않음 |
| --- | --- |
| 팜에 /cache = /rules가 있으며, 요청이 수신됩니다. <b>/content/test.html</b><br><br>/cache<br><br>{<br><br>...<br><br>/rules<br><br>{<br><br>/0001 {<br><br>&quot;allow&quot; 유형<br><br>glob &quot;\*&quot;<br><br>}<br><br>} | 캐시됨 |

<br><br><br><br> <br><br>5. HTTP 요청에 권한 부여 헤더가 없거나 헤더를 포함하고 있지만 /ALLOWAUTHORIZED가 팜 구성에서 1로 설정되어 있습니다.<br><br><br><br> <br><br>
<b>관련 로그 메시지:</b>
<br><br><br><br><br> <br><br> <br><br><br><br>

| `request contains authorization` |
| --- |

<br><br><br><br><br> <br><br>
<b>예:</b>
<br><br><br><br> <br><br>

| HTTP 요청:<br><br>GET /content/test.html<br>  권한 부여: 기본 YWxhZGRpbjpvcGVuc2VzYW1l<br><br>dispatcher.any farm /cache = /allowAuthorized 구성:<br><br>/cache {<br>    /allowAuthorized &quot;0&quot;<br>  } | 캐시되지 않음 |
| --- | --- |
| HTTP 요청:<br><br>GET /content/test.html<br>  권한 부여: 기본 YWxhZGRpbjpvcGVuc2VzYW1l<br><br>dispatcher.any farm /cache = /allowAuthorized 구성:<br><br>/cache {<br>    /allowAuthorized &quot;1&quot;<br>  } | 캐시됨 |

<br><br><br><br> <br><br>6. HTTP 요청은 쿠키 헤더에 로그인 토큰 또는 인증 쿠키를 포함하거나, 해당 쿠키의 중 하나(또는 둘 다)를 포함하며, 팜 구성에서 /ALLOWAUTHORIZED는 1로 설정됩니다.<br><br><br><br> <br><br>
<b>관련 로그 메시지:</b>
<br><br><br><br><br> <br><br> <br><br><br><br>

| `request contains authorization` |
| --- |

<br><br><br><br><br> <br><br>
<b>예:</b>
<br><br><br><br> <br><br>
<b>관련 로그 메시지:</b>
<br><br><br><br> <br><br>
<b>예:</b>
<br><br><br><br> <br><br>

| HTTP 요청:<br><br>GET /content/test.html<br>  쿠키: login-token=...<br><br>dispatcher.any farm /cache = /allowAuthorized 구성:<br><br>/cache {<br>    /allowAuthorized &quot;0&quot;<br>  } | 캐시되지 않음 |
| --- | --- |
| HTTP 요청:<br><br>GET /content/test.html<br>  쿠키: authorization=YWxhZGRpbjpvcGVuc2VzYW1l<br><br>dispatcher.any farm /cache = /allowAuthorized 구성:<br><br>/cache {<br>    /allowAuthorized &quot;0&quot;<br>  } | 캐시되지 않음 |
| HTTP 요청:<br><br>GET /content/test.html<br>  쿠키: login-token=...<br><br>dispatcher.any farm /cache = /allowAuthorized 구성:<br><br>/cache {<br>    /allowAuthorized &quot;1&quot;<br>  } | 캐시됨 |

<br><br><br><br> <br><br>7. URL에 QUERYSTRING 매개 변수가 있으며 팜의 /IGNOREURLPARAMS 구성을 통해 매개 변수를 무시할 수 있습니다.<br><br><br><br> <br><br>
<b>관련 로그 메시지:</b>
<br><br><br><br><br> <br><br> <br><br><br><br>

| `request contains a query string` |
| --- |

<br><br><br><br><br> <br><br>
<b>예:</b>
<br><br><br><br> <br><br>

| HTTP 요청:<br><br>GET /content/test.html?test=1 HTTP/1.1<br><br>팜/캐시 구성:<br><br>/ignoreUrlParams {<br>     /0001 { /type &quot;allow&quot; /glob &quot;\*&quot; } <br>  } | 캐시됨 |
| --- | --- |
| HTTP 요청:<br><br>GET /content/test.html?test=1 HTTP/1.1<br><br>팜/캐시 구성:<br><br>/ignoreUrlParams {<br>     /0001 { /type &quot;deny&quot; /glob &quot;\*&quot; } <br>     /0001 { /type &quot;allow&quot; /glob &quot;test&quot; }<br>  } | 캐시됨 |
| HTTP 요청:<br><br>GET /content/test.html?test=1 HTTP/1.1<br><br>dispatcher.any farm /cache = /allowAuthorized 구성:<br><br>/ignoreUrlParams {<br>     /0001 { /type &quot;deny&quot; /glob &quot;\*&quot; } <br>     /0001 { /type &quot;allow&quot; /glob &quot;q&quot; }<br>  } | 캐시되지 않음 |

<br><br><br><br> <br><br>HTTP 응답 및 캐싱 규칙:<br><br><br><br> <br><br>
다음 기준이 충족되는 경우 AEM에서 반환되는 HTTP 응답은 캐시할 수 있습니다.
<br><br><br><br> <br><br>1. DISPATCHER는 정의된 &quot;RENDERS&quot; 중 하나에서 200 OK 응답을 보내고 받을 수 있습니다.<br><br><br><br><br><br>
참고:

없는 경우 [/timeout](https://helpx.adobe.com/experience-manager/dispatcher/using/dispatcher-configuration.html#DefiningPageRenderersrenders) /renders에 설정되어 있거나 <b>0</b> 그런 다음 인스턴스가 다운된 경우에도 AEM 인스턴스에 대한 연결을 계속 기다립니다.


<br><br><br><br> <br><br>
<b>관련 로그 메시지:</b>
<br><br><br><br><br> <br><br> <br><br><br><br>

| `Unable to send request to remote server.``Unable to receive response from remote server.``Remote server returned: %s``No backend available.` |
| --- |

<br><br><br><br><br> <br><br>
<b>예:</b>
<br><br><br><br> <br><br>

| HTTP 요청:<br><br>200 OK | 캐시됨 |
| --- | --- |
| HTTP 요청:<br><br>404 페이지를 찾을 수 없음 | 캐시되지 않음 |

<br><br><br><br> <br><br>2. 다음 응답 헤더가 응답에 없습니다.<br><br><br><br> <br><br>
- <b>디스패처: 캐시 없음</b>
- <b>캐시 제어: 캐시 없음</b>
- <b>Pragma: 캐시 없음</b>

<br><br><br><br> <br><br>
<b>관련 로그 메시지:</b>
<br><br><br><br><br> <br><br> <br><br><br><br>

| `Backend forbids caching: %s, sent: %s"``response contains no_cache` |
| --- |

<br><br><br><br><br> <br><br>
<b>예:</b>
<br><br><br><br> <br><br>

| HTTP 응답:<br><br>200 OK<br>  디스패처: 캐시 없음 | 캐시되지 않음 |
| --- | --- |
| HTTP 응답:<br><br>200 OK<br>  캐시 제어: 캐시 없음 | 캐시되지 않음 |
| HTTP 응답:<br><br>200 OK<br>  Pragma: 캐시 없음 | 캐시되지 않음 |
| HTTP 응답:<br><br>200 OK | 캐시됨 |

<br><br><br><br> <br><br>3. CONTENT-LENGTH 값이 0바이트보다 큽니다.<br><br><br><br> <br><br>
<b>관련 로그 메시지:</b>
<br><br><br><br><br> <br><br> <br><br><br><br>

| `response content length is zero` |
| --- |

<br><br><br><br><br> <br><br>
<b>예:</b>
<br><br><br><br> <br><br>

| HTTP 응답:<br><br>200 OK<br>  컨텐츠 길이: 0 | 캐시되지 않음 |
| --- | --- |
| HTTP 응답:<br><br>200 OK<br>  컨텐츠 길이: 3120년 | 캐시됨 |
