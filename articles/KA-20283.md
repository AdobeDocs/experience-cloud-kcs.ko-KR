# mixin(및 ClassicUI 사용)이 충돌하여 롤아웃이 실패합니다

## 설명


페이지 롤아웃이 일부 상충으로 인해 실패합니다 *jcr:mixinTypes* 페이지의 축소판 노드에서 다음을 수행합니다.

페이지가 축소판으로 구성되었을 때 충돌이 발생합니다 <b>클래식 UI 사용</b> 그런 다음 처음에 롤아웃됩니다(첫 번째 롤아웃이 작동함).

이 시점에서 페이지의 축소판 구조에서 mixin이 상충되어 마스터 페이지를 롤아웃하는 작업이 다음 중 한 번에 실패합니다.

- dam:축소판
- cq:LiveRelationship

<b>재현 단계:</b>

1. vanilla AEM 6.5.13.0 시작
2. 이동 <b>/siteadmin</b> (사용 <b>클래식 UI</b>)
3. 페이지의 */content/we-retail/language-master/en/women*
4. 이미지 *탭* 이미지를 축소판으로 설정
5. 페이지 롤아웃
6. 이제 <b>사이트 UI</b>
7. 에 대한 페이지 속성 열기 */content/we-retail/us/en/women* 다시
8. 에서 *축소판* 탭에서 *미리 보기 생성* 변경 내용을 저장하고
9. 이제 /siteadmin에서 페이지를 다시 롤아웃합니다.



## 해상도


이 문제는 AEM이 Live Copy 페이지의 jcr:content(/content/we-retail/us/en/women/jcr:content/image/file/jcr:content) 아래에 &quot;dam:thumbnail&quot; 노드를 만들 수 없기 때문에 롤아웃 중에 발생합니다. mixinType &quot;dam:Thumbnail&quot;이 Live Copy 페이지 노드 &quot;/content/we-retail/us/ko/kr/women/jcr:content/image/file/jcr:content&quot; 폴더 아래에 &quot;dam:thumbnail&quot; 폴더를 만들 수 없습니다. &quot;dam:Thumbnail&quot; nodeType 노드 아래에만 이 생성됩니다.

MSM 기본 동작은 롤아웃 0을 사용하는 동안 소스 노드에 추가된 새 mixin을 무시하므로 &quot;/content/we-retail/us/en/women/jcr:content/image/file/jcr:content&quot;에 추가되지 않습니다.

이 문제를 수정하려면 컨텐츠 업데이트 작업 구성을 통해 이 동작을 변경할 수 있습니다.
기본 모든 mixin 유형은 아래 스크린샷에 표시된 대로 무시됩니다.

![](assets/6e22f175-e313-ed11-b83d-002248086a27.png)

다음을 사용하도록 업데이트해야 합니다.


```
(?!.*dam:Thumbnails).*
```
